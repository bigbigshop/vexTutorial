5.1 編程入門與環境配置

前情提要​

機器人現在已具備優異的機械性能，接下來需要透過編程（Programming）為其注入靈魂。在 VEX 體系中，編程不僅是控制馬達轉動，更是實現自動化精準度與戰術邏輯的關鍵。

1. 開發工具選型：積木、官方、還是開源？​

在 VEX V5 的世界裡，根據開發難度與靈活性，主要分為三大流派：

特性VEXcode V5 (Blocks)VEXcode Pro V5 (Text)PROS 4 (進階開源)
程式語言圖形化積木 (Blocks)C++ / PythonC++ (現代化標準)
適用對象完全零基礎、小學/初中隊伍中階隊伍、追求開發穩定性高階隊伍、立志衝擊世界賽
環境配置無需配置，開箱即用內建圖形化設備配置基於 VS Code，需安裝插件
執行效率一般 (解釋執行)高 (編譯執行)極高 (支持實時多工處理)
能力上限難以實現複雜控制演算法適合 80% 的競賽需求(JAR Template/RW-Template)極限性能，支援 LemLib/EZ-Template

2. 第一行程式碼：馬達控制測試​

在 VEX 開發中，首要目標是實現馬達的基礎控制。以下展示將馬達配置於 Port 1 並以 50% 速度前進 的代碼實現。

  • VEXcode Pro V5
  • PROS
/* * VEXcode Pro V5 寫法
 * 註：通常使用 Robot Configuration 工具後，
 * 會在 robot-config.h 自動生成馬達聲明。
 */

#include "vex.h"
using namespace vex;

int main() {
    // 初始化機器人硬體配置
    vexcodeInit();
    
    // 指令格式：motor.spin(方向, 速度, 單位);
    // 使用 50% 的功率百分比讓馬達持續轉動
    leftMotor.spin(forward, 50, percent);
}
/* * PROS 4 寫法
 * PROS 4 推薦使用抽象化程度更高的命名空間與現代 C++ 風格。
 */

#include "main.h"

// PROS 4 建議將設備聲明放在全域或專門的命名空間
pros::Motor left_mtr(1, pros::v5::MotorGears::green); // 需指定齒輪箱顏色/比例

void opcontrol() {
    while (true) {
        //-127 to 127 voltage
        left_mtr.move(127); 

        pros::delay(20);
    }
}

3. 關鍵細節：韌體更新 (Firmware Update)​

這是開發初期最常見的故障原因。若編程軟體無法下載程式，或機器人連接後無反應，通常是主控器版本不一致導致。

同步性要求：V5 主控器（Brain）的內部版本必須與開發工具版本匹配。

操作要領：當螢幕顯示「Firmware Update Needed」時，請務必點擊更新，且過程中切勿拔除數據線。

全鏈路檢查：更新主控器後，務必將遙控器、電池及馬達逐一連接更新，確保整機硬體固件（Firmware）版本同步。

4. 專業開發習慣：版本管理 (Version Control)​

良好的工程習慣能大幅降低比賽風險。

Git 版本控制：每當完成一個穩定的底盤控制或自主程式（Autonomous）邏輯，務必提交（Commit）一次紀錄。

標籤化策略 (Tagging)：在每場比賽前，將當前穩定的程式碼打上標籤（例如 Regional_Final_V1）。

冗餘備份：除了 Git 倉庫，應在隨身碟中存放一份已編譯的穩定版二進位文件（Binary File），以應對開發電腦意外損毀的情況。

🎤 評審面談實戰 (Judges Interview)​

Q1: 你們選用什麼編程平台？原因為何？ (考察重點：工具選型的合理性與團隊需求適配)

🧐 回答範例： 「我們採用 C++ 語言配合 VEXcode Pro V5 平台。選擇 C++ 是因其具備毫秒級的實時控制能力；選用官方平台則是考量其卓越的 API 穩定性，這讓我們能將精力集中在更高階的演算法（如 PID 閉環控制）開發上，而非浪費在處理環境兼容性的問題上。」

Q2: 如果在比賽現場修改程式碼導致錯誤，你們如何處理？ (考察重點：風險管理與工程筆記紀錄)

🧐 回答範例： 「我們嚴格執行版本控制管理。每次重大改動前都會進行 Commit。若現場調校出現 Bug，我們具備隨時回滾（Revert）至上一穩定版本的技術能力。同時，我們隨身攜帶備份程式，確保機器人在決賽中擁有 100% 的可靠性，這也是我們工程流程中關鍵的一環。」
